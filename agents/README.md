# YR 運動研究室 — 代理人小隊說明

## 小隊架構

```
使用者（YRCiou）
    │
    ▼
┌─────────────────────────────┐
│  小隊長 PM（pm.md）          │  ← 解讀需求、拆解任務、整合回報
└─────────────────────────────┘
         │ 發包           ▲ 通過
         ▼                │
┌─────────────────────────────┐
│  工程師 Engineer（engineer.md）│  ← 讀檔、改程式、回報
└─────────────────────────────┘
         │ 完工提交        ▲ 退回修正
         ▼                │
┌─────────────────────────────┐
│  稽查員 Inspector（inspector.md）│  ← 逐項核查、出具報告
└─────────────────────────────┘
```

## 角色分工

| 角色 | 提示詞 | 能力 | 不能做 |
|------|--------|------|--------|
| 小隊長 PM | `agents/pm.md` | 解讀需求、發包、整合 | 不直接改程式碼 |
| 工程師 | `agents/engineer.md` | 讀/寫/改所有檔案 | 不自行 commit/push |
| 稽查員 | `agents/inspector.md` | 讀取所有檔案、核查規範 | 不修改任何檔案 |

## 工作流程

```
1. 使用者說：「我要做 XXX」
        ↓
2. PM 分解任務 → 列出工作清單
        ↓
3. PM 發包給工程師（附任務說明 + CLAUDE.md 背景）
        ↓
4. 工程師執行修改 → 輸出完工報告
        ↓
5. PM 將完工報告轉給稽查員
        ↓
6a. 稽查員通過 → PM 回報使用者，執行 commit/push
6b. 稽查員退回 → PM 重新發包給工程師修正 → 回到步驟 4
```

## 如何啟動小隊

在 Claude Code 對話中，說：

> 「以小隊模式處理：[你的需求]」

Claude 會自動以 PM 角色協調，依序呼叫工程師與稽查員子代理人（Agent 工具）。

## 自動模式（代理人免確認）

代理人執行任務時，所有終端機指令與檔案操作**不需要人工確認**，自動執行並傳送 Telegram 通知。

### 自動放行的條件（任一成立即生效）

| 條件 | 說明 |
|------|------|
| 🌙 夜間排程視窗 | 台灣時間 01:00–03:30（UTC 17:00–19:30），夜間 cron 任務 |
| 🤖 代理人旗標檔 | `%TEMP%/yr-agent-auto.flag` 存在時，所有操作自動放行 |
| 🔗 Telegram Bridge | `TELEGRAM_BRIDGE=1` 環境變數（Bridge 腳本用）|

### 啟用代理人旗標（日間手動任務）

代理人任務開始時**第一行** Bash 加：
```bash
echo 1 > %TEMP%/yr-agent-auto.flag
```
任務結束時**最後一行**清除：
```bash
del %TEMP%/yr-agent-auto.flag
```
旗標管理指令本身永遠自動放行（不會卡關）。

### 排程任務

| 任務 | 時間 | 說明 |
|------|------|------|
| 夜間 SEO/GEO 稽核 | 台灣時間每天 02:00 | 評分、修正、排名查詢、顯示驗證、Telegram 報告 |

## 稽查員審核項目速查

工程師完工後，稽查員會核查以下 9 個面向：
- **A** HTML 結構（Anti-FOUC, canonical, OG, JSON-LD, GTM）
- **B** Header（深/淺切換、搜尋、返回連結）
- **C** 文章 Meta（Badge、作者、日期、瀏覽）
- **D** 版型元件（section-title、lead、表格無直線、配色一致）
- **E** 延伸閱讀（related-posts-section 位置）
- **F** Footer（CC 授權格式）
- **G** Scripts 順序
- **H** posts-data.json 欄位完整性
- **I** 深色模式覆寫
